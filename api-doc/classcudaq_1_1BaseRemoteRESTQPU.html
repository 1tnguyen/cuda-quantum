void cudaq::BaseRemoteRESTQPU::setTargetBackend ( const std::string &  backend)
inlineoverridevirtual

This setTargetBackend override is in charge of reading the specific target backend configuration file (bundled as part of this CUDA-Q installation) and extract MLIR lowering pipelines and specific code generation output required by this backend (QIR/QASM2).

Once we know the backend, we should search for the configuration file from there we can get the URL/PORT and the required MLIR pass pipeline.
